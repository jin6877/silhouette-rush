## 🎮 게임 방법

1. **카메라 켜기** — 브라우저 카메라 권한을 허용하면 전신이 네온 실루엣으로 변신합니다.
2. **자세 맞추기(기준 몸 캘리브레이션)** — 게임 시작 전, **가만히 서 있으면 3·2·1 카운트다운**이 돌고, 그 순간의 **당신의 머리·발·중심(기준 몸 범위)** 을 캡처해 판정 기준으로 잠급니다. 화면엔 **감지된 당신의 머리/발에 스냅되는 가이드선**이 뜨고, 몸 전체가 프레임 안에 들어와 있으면 초록불이 됩니다. 몸이 프레임 밖으로 **잘리면** 카운트다운이 멈추고 *"카메라를 뒤로 옮기거나 기울이세요"* 안내가 뜹니다(디지털 줌은 화각을 못 넓히니 **카메라를 움직이는 게 정답** — 줌은 보조). 기준이 잡히면 **직접 [시작] 버튼**을 눌러 시작합니다(자동 시작 없음 · 다시 맞추기 · 건너뛰기 가능).
3. **구멍에 몸 맞추기** — 무대 안쪽에서 **구멍 뚫린 벽이 다가옵니다.** 벽이 도달하는 순간, 구멍(빈 공간) 모양에 몸을 맞추고 있으면 **통과**, 벽의 채워진 부분에 부딪히면 **충돌**! 이제 **화면 어디에, 얼마나 작게/크게 서 있든** 기준 몸 대비 포즈만 맞으면 통과합니다 — 방 크기·정확한 프레이밍이 더는 필요 없습니다.

라운드가 진행될수록 벽이 더 빨리 오고, 포즈가 더 어렵고 비대칭적이며, 통과 판정도 점점 빡세집니다. 하트 3개를 모두 잃으면 게임 오버.

## ✨ 주요 기능

- 🧍 **실시간 인체 세그멘테이션** — [MediaPipe **Selfie Segmenter**](https://ai.google.dev/edge/mediapipe/solutions/vision/image_segmenter)로 실루엣 마스크를 추출. 영상 전용 경량 모델이라 브라우저에서 30–60fps, **GPU(WebGL) 델리게이트 우선·CPU 폴백**. 추론은 메인 스레드에서 돌지만 모델이 가볍고 **프레임 드롭 + 마스크 시간 보간(EMA)**으로 게임은 항상 60fps로 렌더됩니다.
- 🧭 **몸-상대 판정(핵심 재설계) — "프레임을 사람 몸에 맞춘다"** — 예전엔 플레이어가 자기 몸을 화면의 고정 프레임에 꽉 채워야 포즈가 성립했는데, 디지털 줌은 카메라 화각(FOV)을 못 넓혀 좁은 방에선 머리·발을 동시에 담는 게 물리적으로 불가능했습니다. 이제는 반대로, **판정 직전 플레이어 실루엣을 '자기 기준 몸 범위'로 정규화**해 포즈 구멍 프레임에 대응시킵니다. 결과적으로 **화면 어디에·얼마나 작게/크게·좌우로 치우쳐 서 있든** 기준 몸 대비 포즈만 맞으면 통과합니다.
- ⏳ **카운트다운 캘리브레이션 + 몸에 스냅되는 가이드** — 시작 전 **"가만히 서 있기" 3초 카운트다운**이 끝나는 순간의 머리·발·중심을 평균 내 **기준 몸(Calibration)** 으로 잠급니다. 가이드선은 고정선이 아니라 **감지된 당신의 머리/발에 스냅**되고, 몸이 프레임에 온전히 들어와 있으면 초록, **잘리면 빨강 + "카메라를 뒤로/기울여" 안내**(줌 아님)로 알려주며 카운트다운을 보류합니다. 기준이 잡히면 **[시작] 버튼 활성화**(자동 시작 없음 · 다시 맞추기 가능).
- 🪟 **컨트롤은 카메라 뷰 바깥에** — 줌/오프셋 슬라이더·카운트다운·상태·시작 버튼 등 **모든 컨트롤을 실루엣 뷰포트 밖**(데스크톱=우측 사이드 패널, 좁은 화면=뷰 아래)에 배치해, 맞추는 동안 몸/가이드선이 절대 가려지지 않습니다.
- 🔍 **줌/오프셋은 보조 수단** — **줌 슬라이더(0.5×~2.0×)·상하 위치(±0.3)** 로 프레임을 미세 조정할 수 있지만(세그멘테이션 입력과 화면 실루엣에 동일 적용), 이제 잘림을 고치는 건 **카메라를 물리적으로 움직이는 것**이 정답입니다(디지털 줌은 화각을 못 넓힘). 값은 `localStorage`에 저장됩니다.
- 🎯 **픽셀 겹침 판정(순수 모듈 유지)** — 벽의 채워진 픽셀과 정규화된 플레이어 실루엣의 **겹침 비율**로 통과/충돌을 판정. 겹침 판정 자체는 그대로 두고, **몸-상대 정규화만 순수 전처리로 추가**했습니다.
- 🕺 **20종 포즈 구멍 · 난이도 티어** — 차렷·양팔 벌리기·A자·만세·골대·한 팔 옆·한 팔 번쩍·기울이기·한 손 허리·대각선·가슴 앞 X·점프 스타·옆차기·무릎 올리기·나무·반대 기울이기·웅크리기·깊은 웅크리기·복합 비대칭 등. **좌우 비대칭 포즈**를 적극 활용해 변별력을 높였고, easy→hard 5단계 티어로 라벨링.
- 📈 **점점 어려워지는 난이도 곡선** — 초반은 크고 쉬운 대칭 포즈 + 느린 벽 + 관대한 판정(진입장벽↓), 라운드가 오를수록 **① 벽 접근 속도↑ ② 포즈 티어↑(좁고 비대칭)** ③ **통과 판정 관용도↓**(충돌 허용 0.24→0.11, 요구 채움 0.26→0.34). **셔플백(최근 N개 제외)** 출제로 같은 포즈가 연속으로 나오지 않습니다.
- 💯 **점수 · 콤보 · 하트** — 통과할 때마다 콤보 배수가 붙고, 라운드가 오를수록 난이도 상승.
- 🎇 **네온 게임쇼 연출** — 원근감 있게 밀려오는 벽, 실루엣 글로우(맞으면 라임 그린, 부딪히면 핫핑크), 통과 파티클·충돌 흔들림.
- ⏱️ **3·2·1 카운트다운** — 벽이 다가오는 게이지 + 카운트다운으로 포즈 잡을 여유를 제공.
- 📸 **결과 스코어 카드** — 최종 점수와 마지막 실루엣 스냅샷을 캔버스로 합성해 **이미지 저장 / 클립보드 복사**. 실패 짤 공유용.
- 🔒 **완전 프론트엔드** — 카메라 영상은 서버로 전송되지 않고 **기기 안에서만** 처리됩니다. 서버·API 키 없음.

## 🧠 어떻게 동작하나

```
                    [메인 스레드 · 렌더 60fps]
웹캠 프레임 ─(rVFC로 최신 1장만, 좌우반전 + 줌/오프셋 256×192)─▶ MediaPipe Selfie 세그멘테이션(GPU/WebGL)
      │                        ▲                                   │
      │                        │  추론 중이면 프레임 드롭            ▼
      │                    최신 마스크 ◀───────── 알파 마스크 + 128×96 게임 그리드
      ▼                        │
  네온 실루엣 렌더링 ◀── 마스크 시간 보간(EMA)으로 60fps 부드럽게 채움
                               │
   벽 마스크(포즈 구멍) ─── 겹침 픽셀 판정 ─── 통과 / 충돌 · 점수 · 콤보 · 하트
```

- **메인 스레드에서 GPU(WebGL) 델리게이트로 추론**합니다. MediaPipe wasm 로더는 워커의 `importScripts`에 의존해 ES-module 워커에선 `ModuleFactory not set`으로 실패하는데, 메인 스레드로 옮겨 이 문제를 원천 제거했습니다. 모델이 가벼워 렌더 루프를 실질적으로 막지 않고, GPU 델리게이트도 여기서 안정적으로 붙습니다.
- **최신 프레임만 처리(프레임 드롭)** — `requestVideoFrameCallback`으로 매 웹캠 프레임을 받되, 이전 추론이 진행 중이면 건너뛰어 지연이 쌓이지 않습니다.
- **마스크 시간 보간(EMA)** — 세그멘테이션 업데이트 사이를 지수 이동평균으로 메꿔, 업데이트가 드물어도 실루엣이 끊기지 않고 부드럽게 이어집니다. (판정용 마스크는 반응성 위주의 짧은 시정수.)
- **몸-상대 정규화는 순수 전처리 계층**입니다. 캘리브레이션으로 잡은 기준 몸 `{top, bottom, centerX}` 를 이용해, 매 판정 프레임의 플레이어 마스크를 **닮음변환(균일 스케일 + 평행이동)** 으로 포즈 구멍의 정규 프레임에 워프한 뒤 겹침 판정을 합니다(`warpToCanonical` → `normalizePlayer`). 균일 스케일이라 T자·웅크리기 같은 **포즈 모양이 보존**되고, 판정 순수 함수(`judge`)는 손대지 않았습니다. 캘리브레이션이 없으면(건너뛰기) 항등 변환 = 기존 방식으로 폴백합니다.
- **줌/오프셋은 캡처 전처리 계층**입니다. 웹캠 프레임을 work 캔버스에 그릴 때 중심 기준 스케일 + 오프셋(+좌우반전)으로 변환하므로, 세그멘테이션이 보는 프레임과 화면 실루엣이 **동일한 변환을 타** 일관됩니다. 단, 이건 화각을 넓히지 못하는 보조 수단입니다.
- **입력 소스를 인터페이스로 분리**했습니다. 게임 로직은 `MaskSource`(실제 웹캠) 또는 `FakeMaskSource`(주입형) 어느 쪽으로도 구동됩니다. `FakeMaskSource`로 **작게/구석에 놓인 몸을 주입**해도(카메라 배치 시뮬레이션) 캘리브레이션 정규화 후 포즈만 맞으면 통과함을 헤드리스로 수치 검증합니다.
- **판정·점수·라운드·게임오버 로직은 DOM/모델 없는 순수 모듈**(`src/game/masks.ts`, `src/game/engine.ts`)이라, 합성 마스크로 자동 검증이 가능합니다.

```bash
npx tsx verify/verify.ts   # 구멍에 맞는 마스크=통과, 어긋난 마스크=충돌 등 자동 검증
```

## 🛠️ 기술 스택

- **React 19** + **TypeScript** + **Vite 8**
- **Tailwind CSS v4** (`@tailwindcss/vite`)
- **@mediapipe/tasks-vision** (Selfie Segmenter, 메인 스레드 GPU(WebGL)/WASM, CDN 로드)

## 💻 로컬 실행

```bash
npm install
npm run dev
```

빌드 & 검증:

```bash
npm run build            # 타입체크 + 프로덕션 빌드
npx tsx verify/verify.ts # 게임 코어 로직 자동 검증
```

## 📁 프로젝트 구조

```
src/
├─ game/
│  ├─ masks.ts             # 마스크 유틸 + 20종 포즈 구멍 + 기준 프레임(BODY_FRAME) + 몸-상대 정규화(warpToCanonical/normalizePlayer) + 겹침 판정(순수)
│  ├─ engine.ts            # 게임 상태 머신: 캘리브레이션 정규화·벽 접근·라운드별 판정 강화·셔플백 출제·점수·콤보·하트(순수)
│  ├─ segmentation.ts      # MediaPipe Selfie Segmenter 로드·추론(메인 스레드 GPU→CPU) + 모델 다운로드(진행률)
│  ├─ maskSource.ts        # MaskSource 인터페이스 · WebcamMaskSource(프레임드롭+EMA) · FakeMaskSource
│  ├─ render.ts            # 네온 무대·원근 벽·실루엣·파티클 캔버스 렌더러
│  ├─ scoreCard.ts         # 공유용 결과 카드 캔버스 생성
│  └─ useSilhouetteRush.ts # 게임 루프 훅(소스 주입 + 테스트 훅)
├─ components/             # StartScreen · FramingOverlay(FramingPanel·카운트다운 캘리브레이션 사이드 패널) · Hud · GameOverScreen · LoadingOverlay · PosePreview
└─ App.tsx
verify/verify.ts           # 합성 마스크 기반 헤드리스 검증
```
